

## Persist Applied Recommendations Across Page Refreshes

### Problem

Both TrainingAuditPage and TestResultsModal track applied fixes in React state only (a `Set` or array). On page refresh, that state is lost and all "Apply" buttons become clickable again, even for already-applied recommendations.

### Solution

Store the original recommendation text in the `improvements` table so both pages can load applied state from the database on mount.

### Changes

| File | What |
|------|------|
| Database migration | Add `source_recommendation text` column to `improvements` table. This stores the exact recommendation text used to apply the fix. |
| `supabase/functions/apply-audit-recommendation/index.ts` | Save `source_recommendation` alongside `change_summary` and `patch` when recording the improvement. |
| `src/pages/TrainingAuditPage.tsx` | On agent selection, query `improvements` for that project and load all `source_recommendation` values into `appliedRecs`. Remove the reset-to-empty on agent switch. |
| `src/components/TestResultsModal.tsx` | Extend the existing `fetchApplied` query to also load `source_recommendation` values, adding them to the `appliedFixes` set so audit-originated fixes are also tracked in the University view. |

### Detail

**Edge function change**: When inserting into `improvements`, add `source_recommendation: recommendation` (the raw text string from the request body).

**TrainingAuditPage load logic**:
```text
On selectedAgent change:
  1. Query improvements WHERE project_id = selectedAgent AND source_recommendation IS NOT NULL
  2. Populate appliedRecs Set from the source_recommendation values
```

**TestResultsModal**: The existing fetch already loads `patch` keys. Extend the query to also select `source_recommendation` and add non-null values to the applied set. This ensures fixes applied from Pipeline Audit also show as applied in University, and vice versa.

### Why not just match on `change_summary`?

`change_summary` is generated by AI and may vary in wording. Storing the exact recommendation text used as input gives a reliable, deterministic key for matching.
